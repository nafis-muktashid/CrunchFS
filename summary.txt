# CSE323 Project Summary (CrunchFS-2)

## Current completion

- Phase 1: complete.
- Phase 2: complete.
- Phase 3: complete.
- Phase 4: optional polish only.

## What is now implemented

- Modular codebase:
  - `src/main.cpp`
  - `src/fuse_ops.cpp`
  - `src/metadata.cpp`
  - `src/chunk_store.cpp`
  - `src/thread_pool.cpp`
  - `src/conpresslib.cpp` + `src/compresslib.h`
- Single active compression pipeline: `compresslib` (`conpresslib.cpp`).
- On-disk persistence:
  - `<backing>/.crunchfs/index.txt`
  - `<backing>/.crunchfs/meta/*.meta`
  - `<backing>/.crunchfs/data/*.dat`
- Thread-safe per-file operations with per-file shared/exclusive locks.
- Multithreaded chunk read path using worker pool.
- `.cfs_stats` virtual file endpoint:
  - `logical_bytes`
  - `compressed_bytes`
  - `ratio`

## Supported file operations

- `getattr`, `readdir`, `open`, `create`, `read`, `write`
- `truncate`, `unlink`, `rename`, `utimens`, `chmod`

## Stability fixes completed

- Fixed truncate shrink cleanup to remove correct trailing chunks.
- Added safe handling for zero-length writes.
- Read path now tolerates sparse/missing chunks by returning zero-filled data.
- Guaranteed at least one worker thread even when hardware concurrency reports zero.

## Verification checklist

- Build succeeds (`cmake .. && cmake --build .`).
- Mount and basic CRUD pass.
- Compression is observable (`/dev/zero` vs `/dev/urandom`).
- `.cfs_stats` reports aggregated values.
- Concurrent smoke workload runs against mounted files.
- Hash matches before and after remount.

## One-command validation

- Run `./demo.sh` from project root.
- Script prints PASS/FAIL per section and exits non-zero on failure.

- More FUSE callbacks (`mkdir`, `rmdir`, `chown`, `statfs`, `mknod`, `link`, `symlink`) if required by rubric.
